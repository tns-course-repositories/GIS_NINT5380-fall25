Assignment Step 4 - Table Development

With the Intersection complete, a summary table will be developed and joined to mobile_home_counties. The field FIPS will be utilized within the Group Stat plugin, gaining a summary count of each tornado segment per FIPS county code. This table will then be joined to mobile_home_counties via the FIPS column for further analysis in the attribute table.

Create a new column called Segment and simply provide a value of 1 within the new tornado_county.shp. In effect, this is simply a value per tornado segment that will then be summed per county.

Note

Remember to Toggle OFF the editing mode once the field segment is created and populated with the integer 1 value, and the process is complete.

Next, populate the Group Stat tool as follows, and note the resulting table with tornado counts sum per FIPS. In this step Rows are given the FIPS identifier column and the Value is set to the sum operation for the column titled segment which contains the value 1 for each county tornado segment (a total of 66,947 tornado segments).

With Group Stats still active, navigate to top menu and select to export all records as .csv, pointing to the assignment folder for export and naming the table fips_join:

Assignment Step 5 - Normalization by Geography Attribute

As counties vary significantly in size, the absolute count of tornadoes per county is not yet comparable. To do so, a constant of 100 sq. miles calculated per county, followed by the normalization of counts by geography.

First, Add geometry attributes (area in square meters) within tornado_counties_joined.shp. This will produce a new temporary layer Added geom info with a new area column:

Next, calculate a square miles column with the Field Calculator using the following equation in a new field sq_miles, type decimal. Divide by 100 to normalize size for a final count of tornadoes per 100 sq miles within each county:

(area * 0.00000038610)/100

Finally, normalize the tornado count per 100 county miles within a new field count_100, type whole number:

Note

Note that the resulting column for the segment count is now titled None within Add geom info temporary layer.

Assignment Step 6 - Rank Assignments

In this next step, 4 total rank assignments will be given to a new column risk_rank based on a series of selections.

Use the median societal exposure and tornado incidence scores as the break between high and low exposure and incidence - source.

  • 4 Rank Conditions:

11.22% = Median Mobile Home percentage per County

3.22 Median Tornado segments per 100 miles, per county (when producing this statistic, NULL values are assumed to be value 0.)


Condition #1 -
Low Incidence, Low Exposure
LI = <3.22
LE = <11.22


Condition #2 -
Low Incidence, High Exposure
LI = <3.22
HE = >=11.22


Condition #3 -
High Incidence, Low Exposure
HI = >=3.22
LE = <11.22


Condition #4 -
High Incidence, High Exposure
HI = >=3.22
HE = >=11.22

Utilize fields count_100 and MobileHome for Incidence and Exposure, respectively.

To Start, make each selection condition #1 through #4 within count_100 and MobileHome. Create a new column risk_rank, type whole number and make assignment 1, 2 , 3 and 4 in this new column based on each selection, respectively.

Populate with value -999 to start. Any records after assignments #1 through #4 with -999 resulting are simply the NULL values - counties that have no tornadoes. Those can either be symbolized in the final map as ‘No Incidence Records’:
